# pjt04 (2026-04-17 / 백승우, 장수철)


# 백승우

## 학습한 내용
- 기존 수업 시간에 배웠던 장고 내용을 다시 복습하는 느낌으로 학습했습니다. 처음 프로젝트와 앱을 생성하고, 그에 맞는 조건을 완성한 뒤에 필수 과제를 빠르게 완료할 수 있었습니다. 심화 과정을 통해서 api key를 활용하여 특정 사이트에서 인물 정보를 검색하고 사진을 AI가 가져올 수 있다는 로직을 view.py에서 작성할 수 있다는 것을 학습했습니다.

## 어려웠던 부분
- 필수 학습은 어렵지 않았지만, API KEY를 가져오는 것에서 어려움을 겪었습니다. 단순히 KEY url을 가져오는 것 뿐만이 아니라 base_url도 통일을 시켜야하고, 이에 따라서 LLM과 프롬프트를 여러 차례 수정해가면서 인물에 대한 이름을 검색하였을 때 그에 맞는 인물 정보를 사이트에서 찾아서 시각적으로 나타내는 부분에서 많이 힘들었지만 그래도 구현에 성공해서 다행이었습니다.

## 새로 배운 것들
- bootstrap을 사용해서 웹 사이트를 새로운 형태로 나타낼 수 있고, OPENAI를 사용해서 시각적인 자료도 웹 사이트에 표현할 수 있다는 사실을 새롭게 알게 되었습니다. 사실 필수 학습은 수업 때 배운 내용을 복습하는 느낌이라 새롭게 배우는 느낌은 아니었습니다. 그 중에서는 delete함수를 따로 url을 입력하지는 않았지만, 주소창에 pk/delete/를 쳤을 때 삭제 되는 것도 직접 실행함으로써 배울 수 있었습니다.
- git에서는 main에 기록되어 있는 정보를 현재 내가 develop한 정보와 합쳐서 한꺼번에 업로드하여 바로 수정할 수 있는 코드를 새롭게 알게 되었습니다.
  
## 느낀 점
- 최근에 django를 학습하면서 흥미를 느껴 지속해서 학습해오고 있지만, 아무것도 없는 상태에서 여러 함수를 구현하는 것과, 모든 html코드를 구현하는 것은 여전히 어려운 것 같습니다. 아직 부족하지만 반복적으로 숙달한다면 분명히 익숙하게 구현할 수 있을 날이 올 수 있을 것이라고 생각합니다.

# 장수철

## 학습한 내용
- models.py에서 Movie 모델을 정의하고, title, description, director 같은 필드를 사용해 데이터베이스 구조를 설계하는 방법을 배웠습니다. 마이그레이션이 단순한 명령어가 아니라, 모델의 변경사항을 실제 데이터베이스에 반영하는 과정이라는 점을 이해했다. ModelForm을 활용하여 사용자 입력을 받고, 입력값 검증과 데이터 저장을 효율적으로 처리하는 방법을 학습했습다. views.py에서 index, create, detail, update, delete 함수를 구현하면서 Django의 기본 CRUD 흐름을 익혔습니다.
## 어려웠던 부분
- 템플릿 경로 설정에서 templates/movies/와 movies/templates/movies/ 구조가 혼동되어 TemplateDoesNotExist 오류가 발생했고, 템플릿 위치와 render, extends 경로를 일치시키는 것이 중요하다는 점을 알게 되었습니다.GET 요청과 POST 요청을 각각 다르게 처리해야 한다는 점이 처음에는 헷갈렸습다. 삭제 기능은 단순한 링크가 아니라 POST 요청으로 구현해야 한다는 점이 익숙하지 않았습니다.
## 새로 배운 것들
- get_object_or_404()를 사용하면 특정 데이터를 더 안전하게 조회할 수 있다는 점을 배웠습니다.
Bootstrap의 btn btn-primary, btn btn-danger 같은 클래스는 미리 정의된 스타일을 가져다 쓰는 방식이라는 것을 이해했습니다.
템플릿 상속을 사용하면 공통 UI를 한 곳에서 관리할 수 있어서 훨씬 효율적이라는 것을 배웠습니다.
commit=False를 사용하면 폼에서 저장하기 전에 데이터를 추가로 가공한 뒤 저장할 수 있다는 점을 알게 되었습니다.
## 느낀 점
- Django는 처음에는 구조가 많아서 복잡하게 느껴졌지만, 프로젝트를 하나씩 직접 구현해 보면서 모델, 폼, 뷰, URL, 템플릿이 어떻게 연결되는지 조금씩 이해할 수 있었습니다. 특히 에러가 날 때마다 원인을 하나씩 확인하고 수정하는 과정에서 Django가 어떤 방식으로 동작하는지 더 잘 이해하게 되었습니다.
